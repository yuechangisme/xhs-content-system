# 小红书内容生产流程

## 一、选题与定位

1. 分析小红书当前热门赛道（搜索趋势数据）
2. 结合自身 toolchain 能力筛选可做的方向
3. 确定账号名、视觉风格、内容调性
4. 选定第一篇具体选题

## 二、内容制作

1. **写文案**
   - 标题：15-20 字，核心亮点｜场景标签
   - 正文：150-250 字，朋友式分享
   - 标签：8-12 个（大流量+精准+场景+内容）

2. **找配图**
   - 每种食物配 2-3 张真实实物图
   - 来源优先：Pexels（CDN 直链可用）、国内免版权图站
   - 图片要求：JPEG，宽 ≥600px，无水印
   - 统一命名：`中文名-序号.jpg`（如 `奇亚籽布丁-01.jpg`）

3. **做轮播图 HTML**
   - 尺寸：540×720px（导出 1080×1440，2x 缩放）
   - 色彩：账号品牌色体系
   - 卡片结构：
     - 1/8 封面
     - 2/8 引入/概念
     - 3-6/8 每个食物一卡（顶图+渐变+底部内容）
     - 7/8 总结/公式
     - 8/8 CTA
   - 配图用 `background-image` + `background-size: cover`（兼容 html2canvas）
   - 每张卡不同布局，避免模板感
   - 支持一键下载 ZIP（html2canvas + JSZip + FileSaver）

## 三、本地测试

1. 启动 HTTP 服务：
   ```
   npx http-server -p 8080 content/ --cors
   ```
2. 浏览器打开 `http://127.0.0.1:8080/` 访问 HTML
3. 必须用 `http://` 而非 `file://`（否则 html2canvas 读不到本地图片）

## 四、导出与发布

1. HTML 顶部点"全部下载 (ZIP)"导出 PNG
2. 小红书 APP 上传图片（按顺序）
3. 复制文案发布

## 五、文件归档

```
content/
├── 公共素材库/                          ← 公共图片素材库（每次先来这里找）
│   └── 食物名-序号.jpg
│
└── YYYY-MM-DD-主题关键词/
    ├── 品牌名-主题.html                  ← 轮播图源文件
    ├── 主题-文案.md                     ← 发布文案
    ├── images/                          ← 该篇用图（从公共素材库复制）
    ├── output/                          ← 导出成品
    │   ├── 主题-全部.zip
    │   └── 主题-全部/ (PNG)
    └── drafts/                          ← 过程文件
```
