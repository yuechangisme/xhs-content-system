# 懒人养生手册风格规范

本文档沉淀「懒人养生手册」当前 V6.1 视觉方向，用于后续小红书轮播图生产时保持风格一致。

当前确认可复用模板参考：

`docs/style-reference/lazy-health-v6.1/v6.1-codex-full-7pages.html`

V6.1 的 7 张 PNG 参考图也归档在：

`docs/style-reference/lazy-health-v6.1/`

## 1. 账号视觉基调

- 深绿封面：用于建立账号识别度，适合承载大标题和封面卖点。
- 暖白内容页：用于总览、避坑、方案等信息密度较高的页面，保证手机端阅读舒适。
- 深绿内容页：用于教程页、总结页等需要强化品牌感和收藏感的页面。
- 暖橙强调色：用于编号、重点词、封面卖点图标、CTA 局部强调。
- 深灰正文：用于主要正文信息，避免纯黑带来的压迫感。
- 底部品牌 bar：每页底部保留「懒人养生手册」，弱化显示，不抢正文。

## 2. 固定页面类型

后续轮播图优先从以下页面类型中组合，不随意发明新页面结构：

- 封面页
- 总览原则页
- 教程页
- 避坑对比页
- 方案清单页
- 总结收藏页

## 3. 页面版式骨架

### 封面页

固定骨架：

- 品牌名
- 大标题
- 副标题
- 三个卖点卡
- 底部品牌 bar

使用建议：

- 大标题必须是第一视觉中心。
- 卖点卡使用独立标签结构，不做一条平铺长条。
- 三个卖点应该短、具体、能一眼看懂，例如「省时间」「少纠结」「照着买」。

### 总览原则页

固定骨架：

- 顶部弱品牌栏
- 标题 + 副标题
- 2x2 原则卡
- 下半区速记卡
- 底部品牌 bar

使用建议：

- 2x2 原则卡负责建立整篇内容框架。
- 下半区必须有实际价值，可以是记忆口诀、截图速记、不会选时先看的几条规则。
- 不要只放一句提示，也不要写成长段说明文。

### 教程页

固定骨架：

- 顶部弱品牌栏
- 标题区
- 2x2 主视觉卡
- 3 条说明卡
- 底部提醒
- 底部品牌 bar

使用建议：

- 主视觉卡必须居中，是页面视觉中心。
- 2x2 主视觉卡内每个块包含：轻图标、名称、一句理由、轻标签。
- 3 条说明卡承接主视觉，分别讲「怎么选」「少吃什么」「懒人做法」等单点信息。
- 不允许把内容堆在左上角，也不靠下半屏空白制造呼吸感。

### 避坑对比页

固定骨架：

- 顶部弱品牌栏
- 标题区
- 左右对比卡
- 方法卡
- 底部品牌 bar

使用建议：

- 左右对比卡只做清晰替换关系，例如「少吃什么」和「可以换成」。
- 方法卡给出 2-3 步可执行顺序。
- 页面要像轮播教程卡，不要像文档表格或说明书。

### 方案清单页

固定骨架：

- 顶部弱品牌栏
- 标题区
- 三张厚方案卡
- 高对比 CTA
- 底部品牌 bar

使用建议：

- 方案卡纵向排列，不做横向三栏。
- 每张方案卡包含：方案编号、方案名、具体组合、适合场景。
- 这一页定位是「截图保存购物清单页」，信息必须足够具体。
- CTA 要清晰高对比，不能低对比、不能漂在页面最底部。

### 总结收藏页

固定骨架：

- 顶部弱品牌栏
- 标题区
- 大白总结卡
- 收藏 CTA
- 底部品牌 bar

使用建议：

- 大白总结卡是视觉中心，内容按分类标签组织。
- CTA 和辅助小字要形成整体，不要孤立漂浮。
- 结尾页必须让用户感觉值得保存，而不是简单收尾。

## 4. 字号和可读性规则

- 手机端优先，所有页面按小红书移动端阅读体验判断。
- 正文不能太小，关键文字优先保持 34px 以上。
- 标题必须一眼可读，不能为了塞内容压得过小。
- 信息块内要有清楚层级：标题最强，说明次级，标签最轻。
- 不允许使用大面积无意义空白硬撑页面。
- 不允许用 `flex-1`、`spacer`、`space-evenly` 等方式把内容机械拉满。

### V6.1 QA 字号分级

正式内容如果采用 V6.1 风格，`manifest.json` 应声明：

```json
{
  "styleVersion": "lazy-health-v6.1"
}
```

QA 会按以下层级检查字号，缺少 `styleVersion` 的旧内容继续使用 legacy 规则：

- 主标题 / 封面标题：`.cover-title`，blocking `< 90px`
- 页面标题：`.page-title`，blocking `< 52px`
- 副标题 / CTA 主句：`.subtitle`、`.cta-card .big`、`.save-btn`，blocking `< 34px`
- 主要正文：`.note-value`、`.method-text`、`.sum-value`、`.menu-foods`、`.principle-main`，blocking `< 32px`
- 次级说明：`.food-desc`、`.principle-note`、`.menu-fit`、`.bottom-tip`、`.memory-line`、`.compare-line .muted`，blocking `< 24px`
- 标签 / 弱品牌 / 页码：`.food-tag`、`.brand`、`.topbar .left`、`.topbar .right`、`.cover-offer .hint`，blocking `< 18px`

`.food-desc` 属于次级说明，不按 legacy 正文 46px 阈值阻断；`.note-value`、`.method-text`、`.sum-value` 等主要正文如果过小仍应阻断。

## 5. 文案规则

- 每页一句明确主题。
- 每个信息块只讲一个点。
- 多用短句，少用长段。
- 不写医学功效承诺。
- 不写产品种草话术。
- 不写过度专业术语。
- 语气保持朋友式、懒人式、可执行。
- 表达重点是「照着做更省心」，不是「吃了就会变好」。

## 6. 禁止项

- 禁止重新发明版式。
- 禁止 `flex-1` / `spacer` / `space-evenly` 硬撑页面。
- 禁止内容全部堆在上半屏。
- 禁止低对比 CTA。
- 禁止真人图主导。
- 禁止粗绿外框说明书风。
- 禁止红色大字警示风。
- 禁止文档式长段说明。
- 禁止医疗感、功效承诺、产品种草结构。

可复用关系：

- P3 / P5：教程页模板，适合讲具体搭配、步骤、食物组合、操作方法。

## 8. 极简文字手册风格（V7 实验方向）

从 2026-06-04 起，后续「懒人养生手册」小红书轮播优先采用极简文字手册方向，除非用户明确要求继续使用 V6.1 信息卡片风。

当前确认样例：

`content/视觉实验/minimal-text-handbook-demo/懒人养生手册-极简文字手册样例.html`

对应导出图参考：

`content/视觉实验/minimal-text-handbook-demo/output/极简文字手册样例-v7-完整手册排版-*.png`

### 8.1 整体原则

- 视觉目标是「一本轻手册」，不是信息图、PPT、医学科普海报。
- 信息密度可以高，但视觉密度必须低：少卡片、少图标、少装饰、少模块拼贴。
- 保留品牌色：暖白底 `#FEFAE0`、健康绿 `#3A7D44`、暖橙 `#F4A261`，但绿色和橙色只做文字、编号、高亮和轻强调，不做大面积抢眼装饰。
- 全套页面保持统一：大标题、轻分隔线、淡高亮、底部弱品牌 bar、页码。
- 不使用拼音标题，避免被误认为模仿其他账号。

### 8.2 封面规则

封面固定结构：

1. 顶部小栏目名，例如 `LAZY HEALTH NOTE`
2. 超大中文标题
3. 一句副标题
4. 与主题强相关的生活物件手账插图
5. 底部弱品牌名 + 页码

封面禁止：

- 不放拼音。
- 不放大段解释卡。
- 不把边界提醒、原理说明塞到封面。
- 不使用与主题无关的泛治愈插图。

封面插图优先使用「AI 生成主题插图 + HTML 控制排版」的混合方案：

- 插图应是手账感、纸感、低饱和、生活物件组合。
- 每篇插图必须和主题内容匹配，例如头油用梳子、毛巾、枕套、洗发瓶、温水/茶；熬夜用台灯、水杯、眼罩、月亮；胃胀用小碗、热水杯、散步鞋。
- 插图不写字、不放 logo、不做医疗图标、不做 3D 写实、不做人物主导。
- 插图放在封面下半区，和标题保持紧凑联系，不能上下割裂。

推荐生成提示方向：

```text
warm hand-drawn journal illustration, soft stationery style, simple lifestyle objects related to the topic, cream paper background, muted health green and warm orange palette, clean natural line art, cozy, minimal, no text, no logo, no watermark, no medical symbols, no 3D, no photorealism
```

### 8.3 内容页规则

内容页优先使用「轻分隔线手册页」：

- 顶部小栏目名 + 大标题。
- 下方用淡绿色分隔线组织 2-3 条内容。
- 编号使用轻量暖橙数字，不使用厚圆点编号。
- 每条由「短标题 + 正文解释」组成。
- 标题可以用淡绿色下划高亮，但高亮必须轻，不能像信息图标签。
- 正文保持 34px 左右，保证手机端可读。
- 文案内容本身不强行改成情绪化口语；如果用户确认文案好，只调整排版，不重写内容。

内容页禁止：

- 不做复杂卡片网格。
- 不堆 icon、箭头、流程图。
- 不使用深绿页作为默认内页；深绿只在用户明确需要强品牌感时少量使用。
- 不使用双引号包装重点句，避免 AI 味和模板感。
- 不为了填满页面使用 `flex-1`、`spacer`、`space-evenly`。

### 8.4 总结页规则

总结页继续保持手册感：

- 顶部 `SAVE THIS` + 大标题。
- 用淡分隔线列出 3-5 条可收藏总结。
- 最后放一条边界提醒或行动提醒。
- 边界提醒可以写「如果持续不舒服/明显异常，别硬扛，去看医生」，但不要写医学功效承诺。

### 8.5 与 V6.1 的关系

- V6.1 信息卡片风仍可作为旧内容和特定任务参考。
- 新内容默认优先 V7 极简文字手册风。
- 如果需要正式进入 QA 流程，manifest 的 `styleVersion` 应在 QA 支持该风格后单独声明；在 QA 未支持前，不得随意伪造新的 styleVersion。
